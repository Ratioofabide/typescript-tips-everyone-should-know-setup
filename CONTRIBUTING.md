# Contribution Guidelines

Please note that this project is released with a Contributor Code of Conduct. By participating in this project, you agree to abide by its terms.

## Contents

- [Adding a New Tip](#adding-a-new-tip)
- [Updating Your Pull Request](#updating-your-pull-request)

## Adding a New Tip

Please ensure your pull request adheres to the following guidelines:

- Search open and closed PRs to avoid duplicate suggestions.
- Only submit TypeScript tips that are broadly useful, practical, and educational.
- Each tip should clearly explain:
  - What the technique or feature does.
  - Why it is useful.
  - When it should be used (and any important caveats, if applicable).

- Add your tip to the bottom of the README and add a corresponding link to the table of contents.
- Use concise, descriptive titles in Title Case.
- Prefer examples that:
  - Work with modern TypeScript versions.
  - Demonstrate real-world usage.
  - Are easy to understand without excessive setup.

- Code examples should:
  - Be formatted consistently.
  - Use meaningful variable and type names.
  - Include only the code necessary to demonstrate the tip.
  - Avoid unrelated formatting changes elsewhere in the README.

- If a tip depends on a specific TypeScript version or compiler option, mention it explicitly.
- Make sure your text editor is configured to remove trailing whitespace.
- Check your spelling and grammar.
- The PR should have a clear, descriptive title.

### What Makes a Good Tip?

Good tips generally:

- Teach something that many TypeScript developers may not know.
- Improve type safety, readability, maintainability, or developer experience.
- Highlight useful language features, utility types, compiler options, or best practices.
- Include concise examples that demonstrate the benefit.

Avoid tips that:

- Are overly specific to a single framework or library.
- Duplicate existing tips.
- Primarily reflect personal style preferences without a clear technical benefit.

Thank you for your contribution!

## Updating Your Pull Request

Sometimes a maintainer will ask you to update your pull request before it can be merged. This is usually due to formatting issues, unclear explanations, duplicate content, or because the contribution does not follow the guidelines above.

If you're asked to make changes, simply update your branch and push the new commits to the same pull request.
